# 安全策略 (Security Policy)

## 支持的版本

我们仅对最新 `main` 分支提供安全更新。请基于最新代码报告漏洞。

| 版本 | 是否支持安全更新 |
|:---|:---|
| `main` | ✅ 是 |
| 其他历史提交 | ❌ 否 |

## 报告漏洞

**请不要在公开 issue 中披露安全漏洞。**

请通过以下方式私下报告：

- **邮箱**：向项目维护者发送邮件（kongkong1935@outlook.com），标题注明 `[SECURITY]`。
- 

请在报告中尽量包含：

- 漏洞类型与影响（如信息泄露、任意代码执行等）
- 复现步骤（最小可复现示例 / PoC）
- 受影响版本或 commit
- 建议的缓解或修复方案（如有）

## 响应时限

- **确认收到**：我们会在 **72 小时内** 确认收到报告。
- **初步评估**：在 **7 个工作日内** 给出严重等级与处理计划。
- **修复与披露**：修复完成后随版本发布；公开披露时间会与报告者协商，通常不超过 **90 天**。

## 致谢

对负责任地披露漏洞的贡献者，我们会在修复后于发布说明中致谢（除非你要求匿名）。

## 范围说明

StructFocus 是一个本地优先（local-first）的上下文管理引擎，默认不联网。以下情况通常**不属于**安全漏洞：

- 需要你主动提供 `STRUCT_LLM_API_KEY` 等凭据且凭据本身泄露的场景；
- 你自行部署/分发的派生版本中的问题（请向对应 fork 维护者报告）。

如发现上述范围外但你认为重要的问题，仍欢迎通过上述渠道联系我们。
